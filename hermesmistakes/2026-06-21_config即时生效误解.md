# Config 修改 ≠ 即时生效

> 日期：2026-06-21 · 严重度：🟡 中等 · 模型：deepseek-v4-pro

## 场景

用户要求把 context_length 从 128K 拉到 1M。我执行 `hermes config set model.context_length 1000000` 后说"改完即时生效，下次对话就能用满 1M 上下文了"。

## 错误

说"改完即时生效"——config 写入了，但当前 session 初始化时已经按 128K 开了窗口，这个改不了。正确说法应该是"config 已写入，需要 `/new` 重开 session 或重启 gateway 才能用上 1M"。

## 实际正确答案

用户立刻追问"下次对话？"，纠正了我。config 和 session 是两层：config 决定下次 session 的初始值，本次 session 的窗口大小不可更改。

## 根因

把 config 写入和 session 生效混为一谈。没区分"持久化配置"和"运行时状态"。

## 教训

修改任何 config 后，说明两部分：(1) config 已写，下次 session/重启后生效；(2) 当前 session 不受影响。不要用"即时生效"这种模糊说法。
